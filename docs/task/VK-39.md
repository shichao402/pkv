# VK-39 `note` 可针对 mise 等支持 `conf.d` 的工具提供碎片化落盘方案，减少原子文件冲突

- Vikunja: `PKV#39`
- Status: `执行中`

## 本轮停止条件

- `pkv get <folder> note` 能显式区分默认单文件落盘和 `mise conf.d` 碎片化落盘
- note 可以通过元数据声明稳定的本地目标路径，减少多个来源争抢同一个原子文件
- 新策略有自动化测试覆盖，且不破坏现有单文件同步语义
- README 补齐 note 目标策略和 `mise conf.d` 的使用说明

## 实施步骤

1. 梳理当前 note 落盘逻辑，抽出“远端 note -> 本地目标路径”的解析层
2. 为 Secure Note 增加 note 目标元数据约定，支持默认单文件、`mise conf.d` 策略和显式路径覆盖
3. 在同步流程中接入新的目标解析，保证重命名、冲突预检和状态追踪继续成立
4. 为目标解析和 `mise conf.d` 同步场景补测试
5. 更新 README，说明元数据字段、CLI 用法和适用边界
